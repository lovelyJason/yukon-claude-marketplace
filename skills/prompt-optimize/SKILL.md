# Prompt Optimize - AI 提示词优化助手

分析并优化用户的 Prompt，让 AI 输出更精准、更符合预期。

## 优化维度

### 1. 结构完整性

一个好的 Prompt 应包含：

| 要素 | 说明 | 示例 |
|------|------|------|
| **角色设定** | 告诉 AI 它是谁 | "你是一个资深前端工程师" |
| **任务目标** | 明确要做什么 | "帮我重构这段代码" |
| **上下文** | 提供背景信息 | "这是一个 Vue3 项目，使用 TypeScript" |
| **输入说明** | 描述输入内容 | "我会给你一个组件代码" |
| **输出格式** | 期望的输出形式 | "请用 markdown 格式输出" |
| **约束条件** | 限制和要求 | "不要使用第三方库" |
| **示例** | Few-shot 示例 | "比如输入 X，输出应该是 Y" |

### 2. 清晰度

- 避免模糊表达（"帮我改好一点" → "帮我提升代码可读性"）
- 使用具体数字（"简短一些" → "控制在 100 字以内"）
- 明确动作动词（"处理" → "删除/合并/拆分"）

### 3. 避免歧义

- 专业术语需定义或举例
- 代词指代要明确
- 多步骤任务要编号

### 4. 输出控制

- 指定格式（JSON/Markdown/代码/表格）
- 指定长度（字数/行数/条目数）
- 指定风格（正式/口语/技术文档）
- 指定语言（中文/英文）

## 优化流程

```
1. 接收用户原始 Prompt
2. 分析现有结构和问题
3. 识别缺失要素
4. 提出优化建议
5. 生成优化后的 Prompt
6. 解释优化点
```

## 常见优化模式

### 模式一：补充角色设定

**Before：**
```
帮我写一个登录页面
```

**After：**
```
你是一个资深前端工程师，精通 Vue3 和 TypeScript。

请帮我实现一个登录页面，要求：
- 使用 Vue3 Composition API
- 包含用户名和密码输入框
- 有表单验证（用户名必填，密码至少6位）
- 使用 Element Plus 组件库
- 登录成功后跳转到首页

请直接输出完整的 .vue 文件代码。
```

### 模式二：明确输出格式

**Before：**
```
分析一下这段代码有什么问题
```

**After：**
```
请分析以下代码的问题，按以下格式输出：

## 问题列表

| 行号 | 问题类型 | 问题描述 | 修复建议 |
|------|----------|----------|----------|
| ... | ... | ... | ... |

## 修复后代码

```代码```

## 总结

简要说明主要问题和改进方向（3句话以内）
```

### 模式三：添加约束条件

**Before：**
```
帮我重构这个函数
```

**After：**
```
请重构以下函数，要求：

1. 保持函数签名不变（输入输出类型相同）
2. 提高可读性：
   - 提取魔法数字为常量
   - 减少嵌套层级（不超过2层）
   - 单一职责，必要时拆分辅助函数
3. 添加 JSDoc 注释
4. 不引入新的依赖

请输出重构后的代码，并在注释中说明改动点。
```

### 模式四：Few-shot 示例

**Before：**
```
帮我把这些变量名改成驼峰命名
```

**After：**
```
请将以下变量名转换为小驼峰命名法（camelCase）。

转换规则示例：
- user_name → userName
- get_user_list → getUserList
- MAX_COUNT → maxCount
- UserID → userId

请直接输出转换后的变量名列表，格式：
原名 → 新名
```

## Prompt 模板库

### 代码生成模板

```
你是一个 [技术栈] 专家。

请根据以下需求生成代码：

## 需求描述
[具体需求]

## 技术要求
- 框架：[框架名]
- 语言：[语言]
- 风格：[代码风格要求]

## 输出要求
- 直接输出可运行的代码
- 添加必要的注释
- 处理边界情况

## 示例（可选）
输入：[示例输入]
期望输出：[示例输出]
```

### 代码审查模板

```
请以资深工程师的视角审查以下代码：

## 审查维度
1. 代码正确性：逻辑是否正确
2. 性能：是否有性能问题
3. 安全性：是否有安全漏洞
4. 可维护性：代码是否清晰易读
5. 最佳实践：是否符合 [框架/语言] 最佳实践

## 输出格式
按严重程度分类（🔴 严重 / 🟡 建议 / 🔵 提示），每条包含：
- 问题位置（行号）
- 问题描述
- 修复建议
- 修复代码示例
```

```
[Markdown / JSDoc / TSDoc]

## 要求
- 语言：[中文/英文]
- 包含：功能描述、参数说明、返回值、使用示例
- 风格：[简洁/详细]

## 代码
```代码```
```

## 执行步骤

1. 接收用户的原始 Prompt
2. 检查结构完整性（角色/目标/上下文/格式/约束）
3. 检查清晰度（是否有模糊表达）
4. 检查是否有歧义
5. 生成诊断报告
6. 应用优化模式
7. 输出优化后的 Prompt
8. 解释每处优化的原因

## 注意事项

- 保持用户原意，不要过度改写
- 优化要有针对性，不是越长越好
- 根据使用场景调整复杂度（简单问题不需要复杂 prompt）
- 提供多个版本（简洁版/详细版）供用户选择
- 如果原 Prompt 已经很好，指出优点并给出微调建议
